# Android Inventory Tracker — CS 499 Enhancement One

This directory contains the enhanced Android Inventory Tracker selected for the **Software Design and Engineering** category of CS 499.

## What is included

The public project now includes the complete readable application source and the important Android Studio configuration files:

- `MainActivity.java`
- `model/`: `User` and `InventoryItem`
- `data/`: `DatabaseHelper`, `AuthRepository`, and `InventoryRepository`
- `viewmodel/`: `AuthViewModel` and `InventoryViewModel`
- `util/`: `ValidationUtils` and `PasswordUtils`
- `res/layout/`: login, account creation, inventory, SMS settings, and edit-dialog layouts
- `res/values/`: strings, colors, and day/night themes
- `AndroidManifest.xml`
- Unit tests
- Project and module Gradle configuration
- Gradle version catalog and wrapper configuration
- Enhancement documentation

Generated build output, local machine settings, IDE cache files, and binary launcher images are intentionally excluded from the browsable source because they are not authored application code. They can be regenerated by Android Studio. Visitors can download the repository as a ZIP from GitHub to review the complete published source tree.

## Architecture

The original implementation placed UI construction, navigation, validation, authentication, inventory operations, SQLite schema management, and SMS-permission handling in one `MainActivity.java` file. The enhanced version introduces an MVVM-style separation:

- `model/`: application data objects
- `data/`: SQLite access and repositories
- `viewmodel/`: presentation and business logic
- `util/`: validation and password security
- `MainActivity`: screen coordination and user interaction
- `res/layout/`: separate XML layouts for each screen and dialog

## Security and data improvements

- Salted SHA-256 password hashes replace new plain-text passwords.
- Legacy plain-text passwords are migrated after successful login.
- Parameterized SQLite queries are used for authentication and CRUD operations.
- Inventory rows include an owner value so users access only their own records.
- Input length, format, and numeric-range validation reduces crashes and invalid data.

## Usability improvements

- Full editing for item name, quantity, and location.
- Delete confirmation.
- Clear validation and database-result messages.
- Separate screens for login, account creation, inventory, and SMS settings.

## Testing

`ValidationUtilsTest` verifies valid inventory input, nonnumeric quantities, negative quantities, strong passwords, and short-password rejection.

## Open in Android Studio

1. Download the repository ZIP from GitHub.
2. Extract the files.
3. Open `projects/android-inventory-tracker` in Android Studio.
4. Allow Gradle to synchronize and download required dependencies.
5. Run the app using an Android emulator or compatible device.
