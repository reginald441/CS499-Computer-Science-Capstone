01

Software Design & Engineering

Android Inventory Tracker

In progress

A mobile inventory application supporting registration, login, item creation, item management, and SQLite persistence. The capstone enhancement focuses on restructuring the application into a maintainable architecture rather than making only visual changes.

Enhancement direction

  • Refactor toward an MVVM-style architecture.
  • Separate interface, business logic, and data access.
  • Strengthen input validation and error handling.
  • Improve maintainability, accessibility, and testing.

Professional evidence

ArchitectureMobile UXData persistenceMaintainability

Original course: CS 360 · Mobile Architecture and Programming

02

Algorithms & Data Structures

Corner Grocer

Planned

A C++ application that reads grocery purchase records, counts item frequency, supports searching, and produces reports. The enhancement adds a deeper data-structure comparison and measurable performance analysis.

Enhancement direction

  • Implement a self-balancing red-black tree.
  • Create at least two detailed report formats.
  • Compare tree performance with the original map-based approach.
  • Document runtime, space, scaling, and memory trade-offs.

Professional evidence

C++Red-black treeComplexity analysisReporting

Original course: CS 210 · Programming Languages

03

Databases

Grazioso Salvare Dashboard

Planned

A client/server dashboard that connects a user-facing interface to MongoDB for CRUD operations, filtering, and data visualization. The enhancement concentrates on query performance and secure database interaction.

Enhancement direction

  • Create indexes for frequently queried MongoDB fields.
  • Improve query performance, filtering, and error handling.
  • Validate and sanitize user-supplied values.
  • Reduce NoSQL injection risk and strengthen secure access.

Professional evidence

MongoDBIndexingQuery designSecurity mindset

Original course: CS 340 · Client/Server Development
