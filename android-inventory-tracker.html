Project overview

From a working application to a stronger software product.

The Android Inventory Tracker supports account creation, user login, inventory management, quantity updates, record deletion, SQLite persistence, and SMS-permission handling.

I selected this artifact because it brings together several areas of software development in one application. It demonstrates mobile user-interface design, Java programming, local database integration, validation, security considerations, and the responsibility of maintaining data across multiple screens and user actions.

The original version successfully demonstrated the required functionality, but much of the interface, navigation, business logic, validation, and database access were concentrated in a single activity. The capstone enhancement focused on separating responsibilities and making the project easier to understand, test, extend, and maintain.

Enhancement work

Key software engineering improvements

The milestone enhancement follows the architectural and technical direction developed during the CS 499 code review and instructor feedback.

MVVM-style separation

Organized the application into clearer models, repositories, ViewModels, utilities, interface components, and XML layouts.

Safer authentication

Added salted password hashing and support for migrating legacy plain-text passwords.

User-specific inventory

Associated inventory records with the authenticated user so different accounts do not share the same data.

Improved validation

Strengthened empty-field checks, safer quantity parsing, error messages, and handling of invalid input.

Complete editing

Expanded item management beyond simple quantity increases by supporting meaningful record updates and delete confirmation.

Testing and documentation

Added validation unit tests, an enhancement summary, a README, and testing instructions for review and future maintenance.

Published project structure

Review the implementation, configuration, and supporting work.

The public project includes the Java application source, model, repository, ViewModel, utility, SQLite, manifest, XML layout, theme, test, and Gradle configuration files.

Machine-specific settings, generated build folders, IDE caches, and binary launcher assets are intentionally excluded from the browsable source because they are not authored application logic. Android Studio recreates or downloads those files when the project is opened and synchronized.

Included supporting materials: The enhancement narrative, architecture summary, testing information, and opening instructions are available beside the source code.
